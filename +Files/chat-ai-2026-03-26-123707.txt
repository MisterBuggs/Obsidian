### 🔍 **Why ACO is a Natural Fit for Adaptive DoE**

ACO is a metaheuristic inspired by the foraging behavior of ants, which use pheromone trails to find optimal paths. It excels in:
- **Combinatorial optimization** (e.g., selecting experiment points from a discrete or mixed discrete-continuous space),
- **Exploration vs. exploitation trade-offs** (via pheromone evaporation and reinforcement),
- **Adaptive search** in dynamic or uncertain environments.

These traits align well with the goals of **adaptive DoE**, where:
- The next experiment must be chosen based on prior results,
- The objective function may be noisy or non-linear,
- The search space may be high-dimensional or have hidden constraints.

### 🔄 **How to Integrate ACO into Adaptive DoE**

Here’s a conceptual framework for integrating ACO into an adaptive DoE workflow:

#### 1. **Define the Search Space as a Graph**
- Each node represents a potential experimental condition (e.g., a combination of temperature, pressure, concentration).
- Edges represent transitions between experiments (e.g., small changes in parameters).
- The graph can be fully connected or structured (e.g., based on known physical relationships).

#### 2. **Pheromone Trails Represent "Information Gain" or "Expected Improvement"**
- Pheromone levels on edges are updated based on the **information gain** from previous experiments.
- Information gain can be quantified using:
  - **Prediction variance reduction** (like in D-optimality),
  - **Expected improvement (EI)** in Bayesian optimization,
  - **Model fit improvement** (e.g., decrease in residual sum of squares).

> Example: After running an experiment, if the result significantly improves the model (e.g., reduces uncertainty in a response surface), the path leading to that experiment gets a pheromone boost.

#### 3. **Ants as Experiment Proposers**
- Each "ant" represents a proposal for the next experiment.
- Ants probabilistically choose the next experiment based on:
  - **Pheromone levels** (exploitation of promising regions),
  - **Heuristic information** (e.g., distance to known optima, gradient estimates, or prior model predictions),
  - **Random exploration** (to avoid local optima).

#### 4. **Adaptive Update Rules**
After each experiment:
- Update pheromones on the path taken by the ant(s) that proposed the experiment.
- Use a **decay factor** to prevent premature convergence.
- Optionally, use **global best** or **local best** updates to reinforce high-performing paths.

#### 5. **Integration with Surrogate Models**
- Use a **surrogate model** (e.g., Gaussian Process, Random Forest) to predict outcomes and estimate information gain.
- The surrogate model informs the **heuristic component** of ACO (e.g., expected improvement, entropy reduction).
- This creates a **hybrid ACO–Bayesian Optimization** framework.

2. 100% Neurobasal
3. 40% Ham's F12 + 60% Neurobasal
4. 90% Ham's F12 + 10% Neurobasal

The **next recommendation** would likely include:
- An experiment with significant DMEM (e.g., 60% DMEM + 40% Ham's F12)
- An experiment with DMEM High Glucose (e.g., 50% DMEM High Glucose + 50% Neurobasal)
- Possibly a 100% DMEM experiment
- And one more "refined" mixture based on early results

## Why This Works Better Than Categorical Encoding

With categorical encoding (like one-hot), the algorithm would treat each media as completely separate with no relationship between them. This means:

- It would have no way to know that 70% Ham's F12 + 30% Neurobasal might be similar to 60% Ham's F12 + 40% Neurobasal
- It would waste experiments testing completely arbitrary jumps between media types
- It couldn't discover optimal blends - only pure media types

With the mixture approach, BayBE understands the continuous relationships between media types, so it can:

1. Recognize that similar mixtures should have similar outcomes
2. Use this to predict results for untested mixtures
3. Strategically select new mixtures that maximize information gain

## Practical Evidence from the Literature

This approach is validated in the BayBE paper where they show:

> "With 16 experiments, MORDRED needs less than half the number of iterations compared to OHE [one-hot encoding], which requires 37 experiments."

While they were using chemical descriptors, the same principle applies to your media mixtures - by modeling the continuous relationships between media types, you get to optimal conditions much faster.

## What You'll Actually See in Practice

When you run the optimization:

1. **First batch**: Will include pure media types and simple blends (to establish baseline)
2. **Second batch**: Will start exploring the untested media types (DMEM, DMEM High Glucose)
3. **Subsequent batches**: Will refine the most promising blends, potentially discovering something like "65% Neurobasal + 25% DMEM High Glucose + 10% Ham's F12" as your optimal condition

The algorithm is specifically designed to **explore all media types early on** before focusing on the most promising ones. This is built into the Expected Improvement acquisition function - regions with high uncertainty (like untested media) get prioritized.

## One Important Tip

To ensure it properly explores all media types, make sure your initial data file is empty or only contains a few experiments. If you accidentally include data showing one media type performs extremely poorly, the algorithm might deprioritize it - but this would be biologically appropriate!

The beauty of this approach is that it follows the scientific method: explore broadly first, then focus on the most promising areas - but always maintaining some exploration to avoid missing better options.
